# Portfolio flow

1. Hero: target roles, availability, resume, and project CTA.
2. Experience: chronological proof of work.
3. Projects: concise technical evidence with GitHub path.
4. Skills: quick stack scan.
5. Contact: one clear email action.

## Design principles

- Put recruiter decisions above the fold.
- Use short, scannable evidence rather than dense biography.
- Keep one primary action: open the resume.

## Accessibility requirements

- Semantic headings and landmark sections.
- Keyboard-visible focus states and a skip link.
- Icon-only links have accessible labels.
- Motion respects `prefers-reduced-motion`.
